2 Lista de Comandos

Neste capítulo são descritos os comandos para executar as rotinas, seus parâmetros e modo de usar.

comando: rg-ergbd [OPTION]... FORMAT

rg-ergbd (Region Growing - euclidean RGB distance) ou (Crescimento de Regiões - Distância euclideana em cores RGB).

Esta rotina identifica os tipos de Hematita em 5 classes: Martita, Microcristalina, Granular, Lamelar ou Lobular.

Executa o comando na pasta local ou pasta destino (target DIRECTORY) sobre um par de imagens composto por uma imagem em campo claro (BF - Bright Field) e uma imagem em polarização circular (CPOL). Estas imagens devem ter extensão FORMAT.

Os argumentos em OPTION podem conter os seguintes parâmetros:

-l, --dot-lindex

Opção reservada para programadores, usuários podem desconsiderá-la

-r, --recursive

Executa rg-ergbd de forma recursiva, em todos os diretórios e sub diretórios...

-t, --target-directory=DIRECTORY

Analisa as imagens contidas num diretório específico

-s, --threshold=NUMBER

Limiar para separação dos grãos por diferença de cores entre sementes (Método Analítico). Valor default = 0.032. Este valor já foi identificado como ótimo sobre grandes mosaicos capturados no CTF.

-m, --segmentation-threshold=NUMBER

Limiar de segmentação das hematitas para eliminação de outras fases (goethita, quartzo etc.) (Método Sintético) Valor típico para imagens obtidas na Vale = 0,75 (equivale ao tom de corte de 0,75x256 = 192)

-n, --noise=NUMBER

Ruído acrescentado na imagem para a geração de sementes dos grãos (Método Analítico) Valor default = 0. O uso de ruído ≠0 gera muitas sementes e, apesar de poder melhorar a detecção de cristais, aumenta demais o tempo de processamento.

-c, --canny=NUMBER

Variável do método de Canny utilizado na detecção de bordas (Método Analítico). Valor default = 1.5. Este valor não deverá mudar.

-o, --connectivity=NUMBER

Conectividade (definição de contiguidade dos objetos). Valor default = 4. Não deverá ser alterado.

-i, --scrap-inf=NUMBER

Tamanho mínimo de partículas de hematita (Método Sintético). Valor default = 120 pixels. Partículas menores são eliminadas. Este valor corresponde, aproximadamente, a uma partícula ou cristal com cerca de 10 pixels de diâmetro, ou cerca de 5,3 μm, quando se utiliza a lente de 20X.

-w, --synthetic-method-window-size=NUMBER

Tamanho em pixels da janela de varredura (Método Sintético). Valor default = 50. Este valor já foi identificado como ótimo sobre grandes mosaicos capturados no CTF.

-d, --synthetic-method-window-step=NUMBER

Passo de deslocamento da janela de varredura (Método Sintético). Valor default = 50. Este valor já foi identificado como ótimo sobre grandes mosaicos capturados no CTF.

-g, --synthetic-method-greylevels=NUMBER

Número de níveis de cinza na análise de textura (Método Sintético). Valor default = 32. Este valor provavelmente tem pouco impacto e não deverá ser alterado.

-k, --synthetic-method-tmat-size=NUMBER

Classificador compatível com o tamanho da janela de varredura (w) utilizada. (Método Sintético). Classificador previamente treinado e armazenado, representado por uma Matriz T.

-q, --synthetic-method-classifier-mode='mode'

Tipo de classificador fornecido pelo Octave Linear ou quadrático. Valor default = quadratic.

-p, --particles-minimum-side=NUMBER

Tamanho mínimo de partículas (Método Analítico). Partículas muito pequenas podem gerar erros. Valor default = 10. Partículas com menos de 10 pixels são eliminadas do processo.

-u, --pixel-to-lens-scale-factor=NUMBER

Calibração do pixel em μm. Valor default = 0,53 μm/pixel (para a lente objetiva de 20X e câmera digital MRc).

--help

Mostra esta janela de informação

--version

Mostra informação de versão do código

comando: me-classify [OPTION]...

me-classify (Measure-Classify) ou (Mede-Classifica).

Esta rotina identifica automaticamente o Grau de Maturação de Pelotas.

Executa o comando na pasta local ou pasta destino (target DIRECTORY).

O diretório deve conter os arquivos gerados pelo script Relat Pelota, executado na máquina Windows com o programa AxioVision. São 9 arquivos que devem ter os seguintes nomes (fixos):

  • mosaico-completo.tif
  • mosaico-diametral.tif
  • casca.tif
  • manto-externo.tif
  • manto-interno.tif
  • nucleo.tif
  • distribuicao-poros.tif
  • table-1.csv
  • title.csv

Os argumentos em OPTION podem conter os seguintes parâmetros:

-l, --dot-lindex

Opção reservada para programadores, usuários podem desconsiderá-la.

-r, --recursive

Executa me-classify de forma recursiva, em todos os diretórios e sub diretórios.

-t,--target-directory=DIRECTORY

Analisa as imagens contidas num diretório específico.

-g,--grey-levels=NUMBER

Número de níveis de cinza na análise de textura. Valor default = 256.

-o,--connectivity=NUMBER

Conectividade (definição de contiguidade dos objetos). Valor default = 8.

-s,--threshold=NUMBER

Limiar de segmentação do minério de ferro para a eliminação de outras fases (silicatos, poros, fundo etc.). Valor default = 0.5.

-i,--scrap-inf=NUMBER

Tamanho mínimo de partículas. Valor default = 100.

-j,--scrap-inf-not=NUMBER

Tamanho mínimo de poros. Valor default = 5.

-u,--scrap-sup=NUMBER

Tamanho máximo de poros. Valor default = 3000.

--help

Mostra esta janela de informação

--version

Mostra informação de versão do código
